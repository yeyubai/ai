# Review Checklist Rules

## Frontend
- 组件是否只负责展示，业务编排是否位于 hooks/model
- 页面数据是否在页面顶层拉取并通过 props 下发
- 是否存在跨 feature 直接导入内部文件
- 是否存在重复状态、重复请求、重复 UI 未抽象
- 是否出现大组件/深嵌套条件分支未拆分

## Backend
- Controller 是否仅做入参校验与响应组装
- Service 是否承担业务规则，Repository 是否只做数据访问
- 是否存在事务边界错误（在事务中调用外部网络）
- 是否存在幂等缺失、并发写风险、错误吞没
- DTO/领域模型/响应模型是否明确分离

## Cross-cutting
- 是否新增 `any`、魔法值、语义不清变量名
- 是否补充对应测试（成功路径 + 失败路径）
- 是否更新 API 契约/错误码/迁移文档
- 是否具备可观测性（traceId、结构化日志、关键埋点）

## Design Gate
- 是否存在对应 PRD 需求编号与后端/前端设计文档
- 后端设计是否先于前端设计完成并评审通过
- 编码内容是否与设计文档一致，偏差是否有记录
