## 1. 定位与目标用户

### 1.1 产品定义

本产品不是通用健康记录工具，而是围绕“体重趋势 + 运动执行”展开的执行型减脂驾驶舱。
产品每天只回答三个问题：
- 今天体重状态怎样
- 今天是否完成了应做的运动
- 明天该如何调整得更容易坚持

产品优先解决“先开始、再坚持”的问题，而不是提供完整的饮食、睡眠、社群或训练体系。

### 1.2 核心用户

主目标用户：反复减脂失败者。
典型特征：
- 试过多种方案，但总在中断后放弃
- 愿意记录最关键的数据，但不想被复杂表单绑住
- 对“先执行再解释”的体验更有耐心
- 希望 AI 帮自己把体重变化和运动安排串起来，而不是给长篇理论

### 1.3 服务承诺

我们对用户的承诺是：
- 先围绕体重和运动，把今天最重要的一步做掉
- 用最少输入换最快反馈，不强迫填写无关信息
- 掉队时优先降低门槛，而不是增加羞耻感
- AI 只给可执行动作建议，不给医疗诊断结论
- 饮食与睡眠只作为可选背景信息，不抢占首页主任务

## 2. 业务目标与商业目标

### 2.1 上线后 8 周内业务目标

- G1：D7 留存率 >= 30%
- G2：活跃用户中，当日完成“体重或运动任一核心动作”的占比 >= 65%
- G3：完成晚间 AI 调整的活跃用户占比 >= 45%
- G4：连续 4 周活跃用户中，体重趋势向目标方向变化的用户占比 >= 55%

### 2.2 上线后 8 周内商业目标

- G5：新注册用户中，看到至少 1 次会员价值触点的占比 >= 70%
- G6：会员价值页点击率 >= 12%
- G7：免费用户首个核心动作完成率 >= 80%

### 2.3 指标口径

- 核心动作：`体重记录` 或 `运动完成 / 消耗记录`
- 日闭环完成率：当日完成至少 1 个核心动作且查看首页的活跃用户 / 当日活跃用户
- 晚间 AI 调整触达率：当日进入复盘入口并成功生成调整结果的活跃用户 / 当日活跃用户
- 会员价值触点：在完成核心动作、查看周报、开启高级提醒时出现的权益说明
- 前置指标：`登录后 10 分钟内首个核心动作完成率`、`首页 hero CTA 点击率`、`体重记录成功后返回首页率`、`复盘 ready rate`、`恢复模式进入后 48 小时内回归率`

## 3. MVP 范围

### 3.1 In Scope（必须上线）

- 登录与轻评估：登录、目标体重、减脂周期、运动基线、免费基础版初始化
- 首页双核心闭环：今日下一步、体重状态、运动状态、晚间 AI 调整、失败恢复模式
- 记录中心：体重记录、运动完成/消耗记录、近 7 天补录、饮食/睡眠背景记录
- 进度洞察：体重趋势、运动执行率、周消耗概览、AI 调整回顾
- 会员增强：权益状态、升级触点、文案与埋点占位
- 提醒系统：体重提醒、运动提醒、晚间复盘提醒、静默时段
- 我的与隐私：偏好设置、导出、注销
- H5 登录后全局 App Shell：`首页 / 记录 / 进度 / 我的`

### 3.2 Out of Scope（后续版本）

- 支付与订阅购买链路
- 智能硬件直连
- 社群、挑战赛、好友监督
- 饮食识别和睡眠深度分析
- 增肌 / 维持体重双模式

## 4. 核心闭环

### 4.1 首次使用闭环

1. 用户登录后进入轻评估，补齐目标体重、周期和运动基线
2. 完成评估后初始化免费基础版权益与首日体重/运动任务
3. 自动进入首页，首屏直接给出单一 `nextAction`
4. 用户完成体重记录后，首页切换到运动动作
5. 用户完成运动记录后，晚间进入 AI 调整，形成次日预览

### 4.2 每日重复闭环

1. 首页读取 `GET /api/v1/home/today`，返回体重状态、运动状态、下一步动作、follow-up、恢复模式、会员状态
2. 用户通过首页快捷入口完成体重记录或运动记录
3. 晚间生成 AI 调整，总结当天节奏并给出次日动作建议
4. 次日回到首页时，首页优先呈现新的 `nextAction`

### 4.3 失败恢复闭环

- 连续 2 天未完成体重记录或运动记录时，不惩罚，不清空目标
- 系统切换到恢复模式：降低运动门槛、弱化解释、只给最容易执行的一步
- 恢复模式必须返回 `recoveryMode` 和 `fallbackReason`

### 4.4 商业转化漏斗

- 免费基础版：体重记录、运动记录、基础趋势、基础复盘、次日预览
- 会员增强：更细的动作解释、连续纠偏建议、周报深度洞察、提醒增强
- 升级触点只允许出现在：
- 完成首个核心动作后
- 查看周报或深度洞察时
- 开启高级提醒时

## 5. 信息架构与导航

### 5.1 顶层结构

| Tab | 路由主入口 | 角色定义 | icon（lucide） |
| --- | --- | --- | --- |
| 首页 | `/dashboard` | 今日下一步、体重状态、运动状态、复盘入口 | `House` |

### 5.2 路由归属规则

- `/auth/*`、`/onboarding/*`：不显示 Tab
- `/dashboard`、`/coach/*`：归首页
- `/checkins`、`/checkins/*`：归记录
- `/trend*`：归进度
- `/settings/*`：归我的

### 5.3 首页内容块（固定顺序）

1. `今日下一步` hero
2. `体重状态 + 本周变化 + 连续执行`
3. `体重记录卡 + 运动记录卡`
4. `完成今天后怎么继续`

首页禁止承载：
- 内部术语
- 与当前链路无关的说明卡
- 第二主 CTA
- 饮食、睡眠作为主任务

## 9. 接口与错误码基线

- API 契约主文档：`docs/api/contracts/2026-03-13-api-v2-contracts.md`
- 错误码主文档：`docs/api/errors/2026-03-13-api-v2-error-codes.md`
- 基线要求：
- 路由统一使用 `/api/v1`
- 统一响应结构 `{ code, message, data, traceId }`
- 首页与复盘允许返回降级成功码 `PLAN_FALLBACK_USED`
- 饮食/睡眠保留兼容接口，但不进入首页主任务与首批 AC

## 10. 非功能要求

- 可用性：核心接口月可用性 >= 99.9%
- 性能：`GET /home/today` p95 <= 1200ms；记录接口 p95 <= 400ms；复盘接口 p95 <= 4s
- 安全：敏感字段加密存储；审计日志保留 180 天
- 兼容性：H5 支持最近两个大版本 Chrome/Safari
- 可观测性：首页聚合、运动记录、复盘、提醒发送、会员触点必须具备 tracing 与埋点

## 11. 里程碑与上线门槛

- 第 1 周：PRD / product design / API contracts / RQ 评审冻结
- 第 2 周：`rq-001`、`rq-002`、`rq-003` 开发与联调
- 第 3 周：`rq-004`、`rq-005`、`rq-006`、`rq-007` 开发与联调
- 第 4 周：灰度、埋点校验、回归、放量

放量门槛：
- P0 / P1 缺陷为 0
- 首页空主状态缺陷为 0
- AI 结构化通过率 >= 99%
- 登录与核心动作成功率 >= 99.5%
- 提醒发送成功率 >= 98%
