## 混合 App 本地联调白屏修复
- Summary:
  - 通过 `adb logcat` 定位到 Android WebView 白屏的直接原因并非页面地址不可达，而是首页初始化阶段调用 `POST /api/v1/auth/guest` 时被后端 CORS 拦截，导致根路由因 `sessionStatus=error` 停在空白页。
  - 后端 `main.ts` 改为在开发态除显式配置 origin 外，同时放行 `localhost`、`10.0.2.2`、常见局域网私网 IP 以及 `capacitor://localhost` / `ionic://localhost` 等本地混合联调来源，避免每次切换宿主机 IP 都需要手工改 CORS。
  - 前端根路由 `apps/web/app/page.tsx` 补充“初始化中 / 重试 / 去登录页”的可见兜底界面，即使临时会话初始化失败也不再只剩白屏。
- Files:
  - `backend/src/main.ts`
  - `apps/web/app/page.tsx`
  - `docs/tech/changelog/conversations/2026-03-18.md`
  - `docs/tech/changelog/modules/backend.md`
  - `docs/tech/changelog/modules/frontend.md`
- Notes:
  - 修复后已验证 `OPTIONS http://192.168.2.56:3001/api/v1/auth/guest` 会返回 `Access-Control-Allow-Origin: http://192.168.2.56:3000`；在 `nest --watch` 下后端会自动重启生效。
